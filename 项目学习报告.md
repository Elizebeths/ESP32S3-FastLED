## 1. 项目概述

**FastLED** 是全球最流行的嵌入式可编程LED灯带控制库。简单来说：它让你用一块几块钱的微控制器（如Arduino、ESP32）来驱动成百上千颗彩色LED灯珠，创造出各种炫酷的灯光效果。

一句话总结：**将任何微控制器变成LED灯光 powerhouse**。

## 2. 这个项目在做什么

### 核心任务

FastLED 解决的核心问题是：**如何用微控制器（MCU）精确控制大量可寻址LED灯珠的颜色和亮度**。

可寻址LED（如WS2812B，俗称NeoPixel）是特殊的RGB LED，每颗灯珠内部有控制芯片，可以通过一根数据线**串联**控制成百上千颗灯珠，每颗灯珠独立显示不同颜色。

### 技术原理

FastLED 的工作流程：

```
你的代码设置颜色数组
    → FastLED 将RGB数据转换为芯片特定的时序信号
        → 通过微控制器的GPIO/SPI/I2S/RMT等硬件外设输出精确波形
            → LED灯带接收信号，每颗灯珠显示对应颜色
```

### 典型应用场景

| 场景 | 描述 |

### ⚡ 电源计算

**这是最重要的知识！** 电源不够会烧坏设备或灯光异常。

```
计算公式：
每颗LED最大电流 = 60mA（RGB全白时）
100颗LED × 60mA = 6,000mA = 6A
但是，大多数效果只用 30-50% 的电流
安全系数：取计算值的 80% 左右

示例：
  30颗LED → 最大1.8A → 建议买2.5A电源
  60颗LED → 最大3.6A → 建议买5A电源
  100颗LED → 最大6.0A → 建议买8A电源
  150颗LED → 最大9.0A → 建议买12A电源
  300颗LED → 最大18A → 建议分多路供电
```

### 🔌 接线方法

```
                      ┌─────────────┐
    5V电源(+)  ───────┤ LED灯带 VCC ├──┐
                      └─────────────┘  │
    5V电源(-)  ─┬─────┤ LED灯带 GND │  │
                │     └─────────────┘  │
                │                      │
                │     ┌──────────────┐ │
                └─────┤ MCU GND      │─┘  ← ⚠️ 必须共地！
                      └──────────────┘
                      ┌──────────────┐
    MCU GPIO  ──[220Ω电阻]──┤ LED灯带 DATA │
                      └──────────────┘

    5V电源(+) ──[1000µF电容]── 5V电源(-)   ← 并在电源两端
```

### ⚠️ 安全警告

```
永远不要做的事：
❌ 用MCU的5V引脚给灯带供电（MCU只能提供500mA）
❌ MCU和灯带不共地
❌ 灯带通电时插拔连接线
❌ 一开始就设置最大亮度

一定要做的事：
✅ 30颗LED以上用独立电源
✅ 电源GND同时连接MCU和灯带
✅ 电源两端并联电容
✅ 数据线串接220Ω电阻
✅ 测试时先用低亮度
```


### 验证环境

用这个最简单的代码测试你的环境：

```cpp
#include <FastLED.h>

#define LED_PIN     6    // 数据线接在引脚6
#define NUM_LEDS    10   // 10颗LED
CRGB leds[NUM_LEDS];

void setup() {
    FastLED.addLeds<WS2812, LED_PIN, GRB>(leds, NUM_LEDS);
    FastLED.setBrightness(50);  // 亮度限制50/255
}

void loop() {
    // 红色呼吸效果
    for (int i = 0; i < NUM_LEDS; i++) {
        leds[i] = CRGB::Red;
    }
    FastLED.show();
    delay(500);

    for (int i = 0; i < NUM_LEDS; i++) {
        leds[i] = CRGB::Black;
    }
    FastLED.show();
    delay(500);
}
```

## 9. 核心API速览

### 基本结构

```cpp
#include <FastLED.h>

#define NUM_LEDS 60                           // LED数量
CRGB leds[NUM_LEDS];                         // LED颜色数组

void setup() {
    // 添加LED灯带（核心初始化）
    FastLED.addLeds<WS2812, DATA_PIN, GRB>(leds, NUM_LEDS);
    //                     ↑芯片  ↑引脚  ↑颜色顺序
}
```

### 常用芯片模板

```cpp
// 最常用
FastLED.addLeds<WS2812, DATA_PIN, GRB>(leds, NUM_LEDS);     // WS2812B
FastLED.addLeds<WS2812B, DATA_PIN, GRB>(leds, NUM_LEDS);    // WS2812B别名
FastLED.addLeds<NEOPIXEL, DATA_PIN>(leds, NUM_LEDS);         // NeoPixel别名

// RGBW类型
FastLED.addLeds<SK6812, DATA_PIN, GRB>(leds, NUM_LEDS);     // SK6812

// SPI类型（需要时钟引脚）
FastLED.addLeds<APA102, DATA_PIN, CLOCK_PIN, BGR>(leds, NUM_LEDS);
FastLED.addLeds<DOTSTAR, DATA_PIN, CLOCK_PIN, BGR>(leds, NUM_LEDS);
```

### 颜色定义

```cpp
// RGB颜色（最常用）
leds[0] = CRGB::Red;           // 红色  (255, 0, 0)
leds[0] = CRGB::Green;         // 绿色  (0, 255, 0)
leds[0] = CRGB::Blue;          // 蓝色  (0, 0, 255)
leds[0] = CRGB::White;         // 白色  (255, 255, 255)
leds[0] = CRGB::Black;         // 关闭  (0, 0, 0)

// 自定义颜色
leds[0] = CRGB(255, 128, 0);   // 橙色
leds[0] = CRGB::HotPink;       // 预设颜色
leds[0] = 0xFF8000;            // 十六进制

// HSV颜色（色相/饱和度/亮度）
leds[0] = CHSV(0, 255, 255);   // 红色
leds[0] = CHSV(96, 255, 255);  // 绿色
leds[0] = CHSV(160, 255, 255); // 蓝色

// 颜色转换
CRGB rgb = CRGB(CHSV(hue, 255, 255));  // HSV→RGB
```

### 常用函数

```cpp
// 显示控制
FastLED.show();                          // 将颜色数据发送到LED
FastLED.show(50);                        // 指定全局亮度 0-255
FastLED.setBrightness(128);             // 设置全局亮度
FastLED.clear();                        // 全部熄灭

// 电源管理
FastLED.setMaxPowerInVoltsAndMilliamps(5, 2000);  // 5V, 最大2A
FastLED.setMaxRefreshRate(400);         // 最大刷新率(Hz)

// 颜色工具
CRGB color = blend(CRGB::Red, CRGB::Blue, 128);  // 颜色混合
CRGB faded = color.nscale8(128);                  // 亮度缩放
CRGB inverted = -color;                           // 反色
uint8_t brightness = color.getAverageLight();     // 平均亮度

// 高级
// 并行输出多组LED
// 8路并行 (仅Teensy OctoWS2811)
FastLED.addLeds<OCTOWS2811>(leds, NUM_LEDS_PER_STRIP);

// 后台刷新 (ESP32/Teensy)
FastLED.setMaxRefreshRate(0, true);   // 尽可能快刷新
// FastLED.show() 变为非阻塞，在后台DMA刷新
```

## 13. 常见问题与避坑指南

### 🐛 常见问题速查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 灯不亮 | 接线错误 | 检查GND是否共地，DATA是否接对 |
| 灯乱闪/颜色错误 | 颜色顺序不对 | 尝试GRB/BRG/RGB等不同顺序 |
| 白色偏红/偏蓝 | 电源功率不足 | 换大功率电源，降低亮度 |
| 前几颗灯正常，后面乱 | 数据信号衰减 | 缩短线长，加电平转换 |
| 全白时闪烁 | 电源供电不足 | 降低亮度或换电源 |
| ESP32输出异常 | 3.3V逻辑电平 | 加电平转换模块 |
| 编译错误 | 库路径问题 | 确认库已正确安装 |
| 灯带间歇性闪烁 | 中断干扰 | ESP32用RMT驱动 |

### 💡 Pro Tips

1. **开发时始终用低亮度**（`FastLED.setBrightness(32)`），调试通过后再提高
2. **先点亮1颗LED**，再逐步增加，不要一上来就接100颗
3. **颜色顺序**（RGB vs GRB）是最常见的调试问题，不同品牌灯带可能不同
4. **接地是王道**：90%的问题来自没有正确共地
5. **数据线尽量短**：超过1米建议加电平转换
6. **保留DEMO代码**：把 DemoReel100 设为测试例程，方便验证硬件
7. **利用串口调试**：`Serial.println()` 输出变量值查看程序是否正常运行
